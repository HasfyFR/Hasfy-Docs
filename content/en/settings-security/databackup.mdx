# Data & Backups

This page describes where your data is hosted, how it's protected, the exports available from the interface, and your data protection rights.

## Data location

All Hasfy user and organizational data is hosted on an **OVHcloud** server located in the **Gravelines (GRA) datacenter, France**.

| Component | Hosting | Location |
|-----------|---------|----------|
| PostgreSQL database (self-hosted Supabase) | OVH Gravelines | 🇫🇷 France |
| Authentication (Supabase Auth) | OVH Gravelines | 🇫🇷 France |
| Elasticsearch + Kibana + Fleet (agents) | OVH Gravelines | 🇫🇷 France |
| Backend API (Next.js) | OVH Gravelines | 🇫🇷 France |
| Web interface (frontend) | Vercel CDN | Worldwide (CDN) |

> **GDPR**: all your organizational data (tickets, equipment, users, audit logs) is physically located in France. The web interface (Vercel) does not store user data.

## Encryption

### In transit
All communications between your browser, the monitoring agent, the API, and our servers are encrypted via **HTTPS/TLS**.

### At rest

| Data | Encryption at rest |
|------|--------------------|
| OVH disk volume (PostgreSQL, Elasticsearch…) | ❌ Not encrypted at disk level |
| Tickets, equipment, profiles (PostgreSQL) | ❌ No disk encryption — protected by RLS and network access controls |
| CalDAV passwords and calendar OAuth tokens | ✅ **AES-256-GCM** (application-level encryption) |

Application-level AES-256-GCM encryption applies specifically to calendar connection secrets (CalDAV, Google Calendar, Outlook). The encryption key is stored as a server-side environment variable and is never exposed on the client side.

> **Note**: the absence of disk encryption is compensated by strict access controls: the PostgreSQL server is not exposed to the public internet, access is restricted to the OVH private network, and all tables enforce Supabase **Row Level Security (RLS)** — every query is filtered by `organization_id`.

## Backups

Hasfy uses the automatic backup mechanism provided by self-hosted Supabase:

- **Daily pg_dump**: full export of the PostgreSQL database every night
- **WAL archiving** (Write-Ahead Log): continuous recording of transactions, enabling **point-in-time recovery** (PITR) to any moment within the archiving window

> **RPO / RTO**: recovery point objective and recovery time objective are being finalized with our infrastructure team. Contact us via [Contact](/contactus) for any questions about service continuity.

## Data isolation between organizations

Data isolation between organizations is enforced at two independent levels:

1. **Row Level Security (RLS)**: all Supabase tables automatically filter every query by `organization_id` at the database level
2. **Application-level validation**: all API routes verify the user's membership in the organization before allowing any read or write operation

These two layers are independent: compromising one is not sufficient to expose another organization's data.

## Data exports

Hasfy doesn't (yet) offer a single download center for all your data. Several exports are however available directly from the relevant pages:

| Data | Where to export it | Format |
|--------|---------------|--------|
| **Equipment fleet** | **Equipment** page, "Export equipment" button (requires permission) | CSV / JSON / Markdown |
| **Tickets** | **Support** page, ticket export | CSV / JSON / Markdown |
| **Audit log** | **Settings &gt; Audit**, with severity and date-range filters | CSV / JSON / Markdown |

## Your GDPR rights

Hasfy processes personal data as part of its service and respects the rights provided for under the GDPR: the rights of access, rectification, erasure, portability, and objection.

To exercise any of these rights, contact us via the [Contact](/contactus) page. You can also delete your account directly from **Settings &gt; Account** (see [Account management](/configuration/account)).

## Account or organization deletion

### Organization deletion

The **Owner** role can permanently delete the organization from **Settings &gt; Organization**:

1. Open the danger zone on the Organization page
2. Enter the **exact organization name** to confirm
3. Re-enter your **password**
4. Confirm the deletion

Organization deletion triggers a **cascade across all associated data**: equipment, tickets, messages, members, calendar connections, API tokens, and billing data.

> This action is **irreversible**. If you'd like to keep the organization but change its Owner, use **"Transfer ownership"** from the same page instead.
