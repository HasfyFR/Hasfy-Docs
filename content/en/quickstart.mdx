# Quick Start

This guide walks you through everything from creating your account to getting your dashboard up and running. Plan for about **10 to 15 minutes** to complete all the steps.

## Prerequisites

Before you begin, make sure you have:

- A valid email address (professional or personal)
- A modern web browser (Chrome, Firefox, Safari, Edge)
- Access to a terminal for agent installation *(optional for this initial setup)*

## Step 1: Create your account

1. Go to **[app.hasfy.fr](https://app.hasfy.fr)** and click **"Create an account"**
2. Fill in the registration form: **first name**, **last name**, **email address**, and **password**
3. You can also sign up in one click via **Google**, **GitHub**, or **Microsoft** (OAuth)
4. You are immediately redirected to the onboarding flow

> **Tip**: OAuth sign-in (Google, GitHub, or Microsoft) is the fastest way to get started. You can enable multi-factor authentication (MFA TOTP) at any time from your security settings.

## Step 2: Complete onboarding

Hasfy's onboarding is a **7-step process** that automatically adapts to your profile:

### Step 1 — Calibration questions

The questions depend on your profile:

- **IT Team (owner)**: company size, current ITSM tool, goals
- **MSP**: number of clients managed, RMM/PSA tools used, goals
- **Member**: your role (admin / technician / helpdesk / field), primary use cases, familiarity level

### Step 2 — Organization information

Fill in: organization name, country (France, Belgium, Switzerland, Luxembourg, Monaco, Andorra), city, logo, domain, number of employees, whether a dedicated IT department exists.

### Step 3 — User profile

Complete your first and last name, configure your email notification preferences, and enable dark mode if desired.

### Steps 4 to 7

- **Invitations**: invite members of your team
- **Plan selection**: choose between Starter, Pro, or Max
- **Billing**: enter your payment information
- **Finalization**: your workspace is ready

> **Note**: You can skip invitations and billing during onboarding and come back to them later from the **Admin** tab.

## Step 3: Add your first piece of equipment

From the dashboard, go to the **Equipment** module and click **"+ Add equipment"**. Two modes are available:

### Agent mode (recommended for immediate monitoring)

1. Select **"Via Agent"**
2. A name is automatically generated for the equipment
3. The agent installation flow starts directly (see Step 4)

### Manual mode (inventory without agent)

1. Select **"Manual add"**
2. Fill in the information:
   - **Name**: equipment identifier (e.g. "PC-Office-Martin", "Server-Prod-01")
   - **Type**: laptop, desktop, server, mobile, tablet, printer, network, other
   - **Location** and **organizational unit**
   - **Platform**: Windows, macOS, or Linux
   - **Specifications** *(optional)*: model, OS, CPU, RAM, storage, GPU, IP address, MAC address
   - **Custom fields** *(optional)*: free key-value pairs
3. Click **"Save"**

The equipment appears in your inventory with the status **`not_connected`** (awaiting agent) or **`pending`**.

> **Note**: Manual mode lets you build your inventory without installing an agent. Real-time monitoring (metrics, alerts) requires installing the agent described below.

## Step 4: Install the monitoring agent

The agent (Elastic Agent v9.3.2) is installed via a **command-line script** — there is no graphical installation wizard. The script is pre-configured with your identifiers (organization, equipment, user) and is available directly from the equipment record.

### On Windows (PowerShell)

Open **PowerShell as Administrator** and run the script shown in the equipment record:

```powershell
# Script generated by Hasfy — copy it from the equipment record
Invoke-WebRequest -Uri "<AGENT_URL>" -OutFile elastic-agent.zip
Expand-Archive elastic-agent.zip -DestinationPath .\elastic-agent
cd .\elastic-agent
.\elastic-agent.exe install --fleet-url=<FLEET_URL> --enrollment-token=<TOKEN>
```

### On macOS (Bash)

Open a **Terminal** and run the script shown in the equipment record:

```bash
# Script generated by Hasfy — copy it from the equipment record
curl -O <AGENT_URL>
tar xzf elastic-agent-*.tar.gz
cd elastic-agent-*/
sudo ./elastic-agent install --fleet-url=<FLEET_URL> --enrollment-token=<TOKEN>
```

The macOS script displays colored output and automatically cleans up temporary files.

### On Linux (Bash — universal script)

Open a **terminal** and run the script shown in the equipment record:

```bash
# Script generated by Hasfy — copy it from the equipment record
curl -O <AGENT_URL>
tar xzf elastic-agent-*.tar.gz
cd elastic-agent-*/
sudo ./elastic-agent install --fleet-url=<FLEET_URL> --enrollment-token=<TOKEN>
```

> **Important**: Linux installation uses only the **universal script** (`.tar.gz` archive). There is no `.rpm` package. This script is compatible with all supported Linux distributions.

### Verification

After installation, return to the equipment record in Hasfy:

- The status transitions from **`not_connected`** → **`pending`** → **`up`** (equipment online)
- Detailed metrics (CPU, RAM, disk, network) are available within the equipment record

## Step 5: Explore the dashboard

Go to the **Dashboard** page. It displays **4 cards**:

| Card | Content |
|------|---------|
| **Support** | Last ticket (title, status, priority, assigned technician) + "New ticket" button |
| **Equipment** | Carousel of your devices + "Add equipment" button |
| **Alerts** | Critical and Warning counters + last alert message + "View all" link |
| **Overview** | Open tickets, critical tickets, total equipment + links to modules |

> **Note**: Real-time metrics (CPU, RAM, disk) and health graphs are available in the **detail record** of each piece of equipment, not on the main dashboard page.
