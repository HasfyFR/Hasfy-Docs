# Onboarding : Configuration initiale

Lors de votre première connexion à Hasfy, un parcours d'onboarding vous guide à travers la configuration de votre espace de travail. **Le flux s'adapte automatiquement à votre rôle** : le créateur d'une organisation suit un parcours Owner (5 à 6 étapes), tandis qu'un membre invité suit un parcours allégé de 3 étapes.

---

## Création du compte

Accédez à **[app.hasfy.fr/auth/signup](https://app.hasfy.fr/auth/signup)** et remplissez le formulaire d'inscription :

- **Prénom** *(obligatoire)*
- **Nom** *(obligatoire)*
- **Adresse email** *(obligatoire)*
- **Mot de passe** *(obligatoire)* — la visibilité peut être activée via le bouton dédié

Vous pouvez aussi créer votre compte en un clic via **Google**, **GitHub** ou **Microsoft** (OAuth).

> **Astuce** : Si vous avez reçu un lien d'invitation contenant le paramètre `?plan=`, le plan correspondant (Starter, Pro ou Max) sera présélectionné automatiquement à l'étape d'abonnement.

---

## Flux Owner — Créateur d'organisation

Ce flux s'applique à toute personne qui crée une nouvelle organisation Hasfy. Le créateur obtient automatiquement le rôle **Owner** (non révocable, mais transférable).

### Étape 0 — Calibration

La première chose à faire est de choisir votre **type d'organisation** entre deux options présentées sous forme de cartes visuelles :

- **IT Team** : équipe IT interne d'une entreprise
- **MSP** : prestataire de services managés gérant plusieurs clients

Ce choix détermine les questions suivantes et conditionne l'affichage de l'étape de facturation MSP.

#### Questions IT Team (3 questions)

1. **Taille de l'entreprise**
   `1-10` | `11-50` | `51-250` | `250+`

2. **Solution actuelle**
   - Rien / ad hoc
   - Tableurs
   - Équipe IT interne
   - Outil existant *(saisie texte libre si sélectionné)*

3. **Objectif principal** *(avec icônes)*
   - Surveiller la santé des appareils
   - Renforcer la sécurité
   - Simplifier le support employé
   - Centraliser l'inventaire
   - Autre *(saisie texte libre)*

#### Questions MSP (3 questions)

1. **Nombre de clients gérés**
   `1-5` | `6-20` | `21-50` | `50+`

2. **Outils actuels**
   - Aucun
   - RMM uniquement
   - PSA uniquement
   - RMM + PSA
   - Autre *(saisie texte libre)*

3. **Objectif principal**
   - Gérer à l'échelle
   - Automatiser
   - Sécuriser
   - Centraliser
   - Autre *(saisie texte libre)*

> **Note** : Les réponses à la calibration sont sauvegardées en base. Si vous revenez sur le parcours, elles ne vous sont pas redemandées.

---

### Étape 1 — Informations sur l'organisation

Renseignez les informations de votre organisation :

| Champ | Obligatoire | Détail |
|-------|:-----------:|--------|
| Logo | Non | Upload image — bouton de suppression visible au survol |
| Nom de l'organisation | **Oui** | Apparaît dans l'interface et les communications |
| Pays | **Oui** | 6 options uniquement : France 🇫🇷, Belgique 🇧🇪, Suisse 🇨🇭, Luxembourg 🇱🇺, Monaco 🇲🇨, Andorre 🇦🇩 |
| Ville | **Oui** | Saisie texte libre |

Le bouton **Continuer** reste désactivé tant que le nom, le pays et la ville ne sont pas renseignés.

> **Important** : Le créateur du compte devient automatiquement **Owner** de l'organisation. Ce rôle confère tous les droits d'administration. Il ne peut pas être révoqué mais peut être transféré à un autre membre.

---

### Étape 2 — Membres de l'équipe

Invitez les membres de votre équipe avant de finaliser le parcours. Deux méthodes sont disponibles :

#### Ajout manuel

Pour chaque membre, renseignez :
- Email, Prénom, Nom, Titre de poste, Rôle

#### Import CSV

Téléchargez le template fourni et importez un fichier avec les en-têtes suivants :

```
firstName, lastName, email, job, role
```

Les erreurs de validation indiquent le **numéro de ligne** concerné.

#### Rôles assignables

| Rôle | Libellé affiché | Limite |
|------|----------------|--------|
| `user` | Employé | — |
| `technician` | Technicien | — |
| `admin` | Admin | **2 admins maximum** par organisation |

> **Note** : Chaque organisation est limitée à **250 utilisateurs**. Vous pouvez cliquer sur **« Faire ça plus tard »** pour passer cette étape et inviter des membres depuis la page **Utilisateurs & groupes**.

---

### Étape 3 — Facturation (MSP uniquement)

Cette étape s'affiche **exclusivement** pour les organisations de type MSP.

Une seule option est disponible : **Facturation manuelle** (`msp_billing_mode = "manual"`).

---

### Étape 3 ou 4 — Abonnement

Hasfy propose un plan unique affiché avec un badge **Beta** :

| | Tarif |
|-|-------|
| Plan de base | **150 €/utilisateur/mois** ~~200 €~~ |
| Add-on IA *(optionnel)* | +30 €/siège/mois |

Le nombre de sièges est calculé automatiquement :

```
Sièges = max(1, nombre de membres ajoutés + 1)
```
*(l'Owner est inclus dans le décompte)*

Le paiement est traité via **Mollie** — vous êtes redirigé vers une page de checkout externe. Au retour sur Hasfy (`?payment_return=true`), les **invitations sont envoyées automatiquement** à tous les membres ajoutés à l'étape précédente.

---

### Étape finale — Completion

Un écran animé confirme la finalisation de votre configuration avec un résumé en 3 éléments :

1. **Inviter des membres** — marqué complété si au moins un membre a été ajouté
2. **Configurer la sécurité** — toujours marqué complété
3. **Facturation** — toujours marqué complété

Cliquez sur **« Accéder au tableau de bord »** pour être redirigé vers `/app`.

## Flux Member — Membre invité

Ce flux s'applique à toute personne rejoignant une organisation déjà existante via une invitation.

### Étape 0 — Calibration membre

3 questions pour personnaliser votre expérience :

1. **Votre rôle dans l'équipe**
   Admin | Technicien | Helpdesk | Technicien terrain

2. **Usage principal**
   Monitoring | Tickets | Reporting | Tout

3. **Familiarité avec les outils RMM**
   Débutant | Quelques bases | Expert

---

### Étape 1 — Profil

Vérifiez ou complétez vos informations personnelles :

- **Prénom** *(obligatoire)* — pré-rempli depuis l'authentification
- **Nom** *(obligatoire)* — pré-rempli depuis l'authentification

Deux préférences à configurer :

| Préférence | Valeur par défaut |
|------------|:-----------------:|
| Notifications email | Activé |
| Mode sombre | Désactivé |

---

### Étape 2 — Notifications

Affinez vos préférences de notifications avec 4 toggles :

| Notification | Valeur par défaut |
|--------------|:-----------------:|
| Nouveaux tickets | Activé |
| Mises à jour des tickets | Activé |
| Alertes critiques | Activé |
| Rapport hebdomadaire | Désactivé |

---

### Étape finale — Completion membre

L'écran final affiche 3 indicateurs de statut **en ligne** (points verts) :

- Monitoring client
- Gestion des tickets
- Notifications

Cliquez sur **« Aller au tableau de bord »** pour être redirigé vers `/app`.

---

## Modification ultérieure

Les informations renseignées durant l'onboarding sont modifiables depuis les paramètres de l'application :

| Information | Où la modifier |
|-------------|----------------|
| Profil personnel (prénom, nom) | Paramètres › Mon profil |
| Logo et nom de l'organisation | Paramètres › Organisation |
| Membres de l'équipe | [Utilisateurs & groupes](/core-features/usermanagement) |
| Préférences de notifications | Paramètres › Notifications |
| Mode sombre | Paramètres › Apparence |
