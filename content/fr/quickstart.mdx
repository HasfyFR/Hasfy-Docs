# Quick Start : Démarrage rapide

Ce guide vous accompagne de la création de votre compte jusqu'à la prise en main de votre tableau de bord. Comptez environ **10 à 15 minutes** pour compléter l'ensemble des étapes.

## Prérequis

Avant de commencer, assurez-vous de disposer de :

- Une adresse email valide (professionnelle ou personnelle)
- Un navigateur web moderne (Chrome, Firefox, Safari, Edge)
- Un accès à un terminal pour l'installation de l'agent *(optionnel pour cette première prise en main)*

## Étape 1 : Créer votre compte

1. Rendez-vous sur **[app.hasfy.fr](https://app.hasfy.fr)** et cliquez sur **« Créer un compte »**
2. Remplissez le formulaire d'inscription : **prénom**, **nom**, **adresse email** et **mot de passe**
3. Vous pouvez également vous inscrire en un clic via **Google**, **GitHub** ou **Microsoft** (OAuth)
4. Vous êtes aussitôt redirigé vers le parcours d'onboarding

> **Astuce** : La connexion via OAuth (Google, GitHub ou Microsoft) est la méthode la plus rapide pour démarrer. Vous pouvez activer l'authentification multi-facteurs (MFA TOTP) à tout moment depuis vos paramètres de sécurité.

## Étape 2 : Compléter l'onboarding

L'onboarding Hasfy est un parcours de **7 étapes** qui s'adapte automatiquement à votre profil :

### Étape 1 — Questions de calibration

Les questions posées dépendent de votre profil :

- **IT Team (propriétaire)** : taille de l'entreprise, outil ITSM actuel, objectifs
- **MSP** : nombre de clients gérés, outils RMM/PSA utilisés, objectifs
- **Membre** : votre rôle (admin / technicien / helpdesk / terrain), usages prioritaires, niveau de familiarité

### Étape 2 — Informations sur l'organisation

Renseignez : nom de l'organisation, pays (France, Belgique, Suisse, Luxembourg, Monaco, Andorre), ville, logo, domaine, nombre d'employés, présence d'un département IT dédié.

### Étape 3 — Profil utilisateur

Complétez vos prénom et nom, configurez vos préférences de notifications email et activez le mode sombre si vous le souhaitez.

### Étapes 4 à 7

- **Invitations** : invitez les membres de votre équipe
- **Sélection du plan** : choisissez entre Starter, Pro ou Max
- **Facturation** : renseignez vos informations de paiement
- **Finalisation** : votre espace de travail est prêt

> **Note** : Vous pouvez ignorer les invitations et la facturation lors de l'onboarding et y revenir plus tard depuis l'onglet **Admin**.

## Étape 3 : Ajouter votre premier équipement

Depuis le tableau de bord, accédez au module **Équipements** et cliquez sur **« + Ajouter un équipement »**. Deux modes sont disponibles :

### Mode Agent (recommandé pour une supervision immédiate)

1. Sélectionnez **« Via Agent »**
2. Un nom est généré automatiquement pour l'équipement
3. Le flux d'installation de l'agent démarre directement (voir Étape 4)

### Mode Manuel (inventaire sans agent)

1. Sélectionnez **« Ajout manuel »**
2. Remplissez les informations :
   - **Nom** : identifiant de l'équipement (ex. « PC-Bureau-Martin », « Serveur-Prod-01 »)
   - **Type** : laptop, desktop, serveur, mobile, tablette, imprimante, réseau, autre
   - **Localisation** et **unité organisationnelle**
   - **Plateforme** : Windows, macOS ou Linux
   - **Spécifications** *(optionnel)* : modèle, OS, CPU, RAM, stockage, GPU, adresse IP, adresse MAC
   - **Champs personnalisés** *(optionnel)* : paires clé-valeur libres
3. Cliquez sur **« Enregistrer »**

L'équipement apparaît dans votre inventaire avec le statut **`not_connected`** (en attente d'agent) ou **`pending`**.

> **Note** : Le mode manuel permet de construire votre inventaire sans installer d'agent. La supervision en temps réel (métriques, alertes) nécessite l'installation de l'agent décrite ci-dessous.

## Étape 4 : Installer l'agent de supervision

L'agent (Elastic Agent v9.3.2) est installé via un **script en ligne de commande** — il n'existe pas d'assistant graphique. Le script est pré-configuré avec vos identifiants (organisation, équipement, utilisateur) et est disponible directement dans la fiche de l'équipement.

### Sur Windows (PowerShell)

Ouvrez **PowerShell en tant qu'administrateur** et exécutez le script affiché dans la fiche équipement :

```powershell
# Script généré par Hasfy — copiez-le depuis la fiche équipement
Invoke-WebRequest -Uri "<URL_AGENT>" -OutFile elastic-agent.zip
Expand-Archive elastic-agent.zip -DestinationPath .\elastic-agent
cd .\elastic-agent
.\elastic-agent.exe install --fleet-url=<FLEET_URL> --enrollment-token=<TOKEN>
```

### Sur macOS (Bash)

Ouvrez un **Terminal** et exécutez le script affiché dans la fiche équipement :

```bash
# Script généré par Hasfy — copiez-le depuis la fiche équipement
curl -O <URL_AGENT>
tar xzf elastic-agent-*.tar.gz
cd elastic-agent-*/
sudo ./elastic-agent install --fleet-url=<FLEET_URL> --enrollment-token=<TOKEN>
```

Le script macOS affiche une sortie colorée et effectue un nettoyage automatique des fichiers temporaires.

### Sur Linux (Bash — script universel)

Ouvrez un **terminal** et exécutez le script affiché dans la fiche équipement :

```bash
# Script généré par Hasfy — copiez-le depuis la fiche équipement
curl -O <URL_AGENT>
tar xzf elastic-agent-*.tar.gz
cd elastic-agent-*/
sudo ./elastic-agent install --fleet-url=<FLEET_URL> --enrollment-token=<TOKEN>
```

> **Important** : L'installation Linux utilise uniquement le **script universel** (archive `.tar.gz`). Il n'existe pas de paquet `.rpm`. Ce script est compatible avec toutes les distributions Linux supportées.

### Vérification

Après l'installation, retournez sur la fiche équipement dans Hasfy :

- Le statut passe de **`not_connected`** → **`pending`** → **`up`** (équipement en ligne)
- Les métriques détaillées (CPU, RAM, disque, réseau) sont disponibles dans la fiche équipement

## Étape 5 : Explorer le tableau de bord

Rendez-vous sur la page **Tableau de bord**. Il présente **4 cartes** :

| Carte | Contenu |
|-------|---------|
| **Assistance** | Dernier ticket (titre, statut, priorité, technicien assigné) + bouton « Nouveau ticket » |
| **Équipements** | Carrousel de vos appareils + bouton « Ajouter un équipement » |
| **Alertes** | Compteurs Critique et Warning + dernier message d'alerte + lien « Voir tout » |
| **Vue d'ensemble** | Tickets ouverts, tickets critiques, total équipements + liens vers les modules |

> **Note** : Les métriques temps réel (CPU, RAM, disque) et les graphiques de santé sont disponibles dans la **fiche détaillée** de chaque équipement, pas sur la page d'accueil du tableau de bord.
