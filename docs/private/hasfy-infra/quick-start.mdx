# Guide de démarrage rapide - GCP Phase 1

Déployez Elasticsearch **Phase 1** (3 nodes multi-rôle) sur GCP en **30 minutes**.

**Coût** : 381€/mois | **Capacité** : 100 agents, 50 GB/jour

---

## 🎯 Qu'allez-vous déployer ?

**Architecture Phase 1** :
- ✅ 3 nœuds Elasticsearch multi-rôle [master, data, ingest]
- ✅ 1 Kibana (avec auto-scale jusqu'à 2 replicas)
- ✅ 2 Fleet Server
- ✅ Load Balancer multiplexé (1 IP)
- ✅ Backups quotidiens GCS
- ✅ SSL/TLS automatique

**Quand migrer vers Phase 2** : >150 agents OU >75 GB/jour

## ⚡ Installation express

### 1. Prérequis (5 min)

```bash
# Installer les outils nécessaires
brew install terraform kubectl google-cloud-sdk jq

# Se connecter à GCP
gcloud auth login
gcloud auth application-default login

# Définir le projet
gcloud config set project YOUR_PROJECT_ID

# Activer les APIs nécessaires
gcloud services enable \
  container.googleapis.com \
  compute.googleapis.com \
  storage.googleapis.com
```

### 2. Configuration Phase 1 (5 min)

```bash
# Naviguer vers le provider GCP
cd providers/gcp

# Configurer Terraform
cd terraform
cp terraform.tfvars.example terraform.tfvars

# Éditer les variables
nano terraform.tfvars
```

**Variables Phase 1 (minimum requis)** :
```hcl
# Projet GCP
project_id  = "your-project-id"
region      = "europe-west9"  # Paris
zone        = "europe-west9-a"

# Domaine (optionnel - peut utiliser IP)
base_domain = "hasfy.fr"

# Phase 1 : Architecture multi-rôle
deployment_phase = "phase1"  # Phase 1 par défaut

# Elasticsearch Phase 1 : 3 nodes multi-rôle
elasticsearch_node_count = 3
elasticsearch_node_roles = ["master", "data", "ingest"]
elasticsearch_cpu        = "2000m"
elasticsearch_memory     = "8Gi"
elasticsearch_storage    = "100Gi"

# Kibana Phase 1
kibana_replicas = 1

# Fleet Server Phase 1
fleet_replicas = 2

# Environnement
environment = "production"
```

### 3. Déploiement automatique (20 min)

```bash
# Retour au dossier scripts
cd ../scripts

# Rendre le script exécutable
chmod +x deploy.sh

# Lancer le déploiement Phase 1
./deploy.sh
```

**Le script va** :
1. ✅ Vérifier les prérequis (gcloud, kubectl, terraform)
2. ✅ Déployer GKE Autopilot (5 min)
3. ✅ Créer les ressources réseau (VPC, subnets)
4. ✅ Réserver 1 IP Load Balancer
5. ✅ Installer ECK Operator (2 min)
6. ✅ Déployer 3 nœuds Elasticsearch Phase 1 (8 min)
7. ✅ Déployer Kibana (3 min)
8. ✅ Déployer Fleet Server (2 min)
9. ✅ Configurer DNS et SSL
10. ✅ Configurer Fleet Server automatiquement
11. ✅ Afficher credentials

**⏱️ Temps total** : 20-25 minutes

```bash

# kibana-kb-xxx                       1/1     Running   0          8m
# fleet-server-agent-xxx              1/1     Running   0          6m
# fleet-server-agent-yyy              1/1     Running   0          6m

# Doit afficher :
# NAME            HEALTH   NODES   VERSION   PHASE   AGE
# elasticsearch   green    3       8.11.0    Ready   10m
```

### 5. Récupérer les credentials (1 min)

```bash
# Récupérer le mot de passe Elasticsearch
kubectl get secret elasticsearch-es-elastic-user \
  -n elastic-system \
  -o go-template='{{.data.elastic | base64decode}}'

# Sauvegarder (IMPORTANT)
echo "Elasticsearch password: [VOTRE_MOT_DE_PASSE]" > .credentials.txt

# Récupérer l'URL du Load Balancer
kubectl get ingress -n elastic-system

# Ou via Terraform
cd ../terraform
terraform output load_balancer_ip
```

**Se connecter à Kibana** :

**Option 1 - Avec domaine configuré** :
1. Ouvrir `https://kibana.hasfy.fr` (ou votre domaine)
2. Username : `elastic`
3. Password : (celui récupéré ci-dessus)

**Option 2 - Sans domaine (test local)** :
```bash

kubectl port-forward -n elastic-system svc/kibana-kb-http 5601:5601 &

# Ouvrir https://localhost:5601
# Username: elastic
# Password: (celui récupéré)

# Arrêter port-forward
pkill -f "port-forward.*5601"
```

## 🆘 Troubleshooting Phase 1

### Pods ne démarrent pas

```bash
# Vérifier events
kubectl get events -n elastic-system --sort-by='.lastTimestamp' | tail -20

# Vérifier logs Elasticsearch
kubectl logs -n elastic-system elasticsearch-es-default-0 --tail=100

# Vérifier ressources disponibles (GKE Autopilot scale automatiquement)
kubectl describe pod -n elastic-system elasticsearch-es-default-0
```

**Cause fréquente** : GKE Autopilot en cours de provisioning de nodes. **Attendre 5-10 minutes**.

### Cluster health YELLOW ou RED

```bash
# Diagnostic complet
kubectl port-forward -n elastic-system svc/elasticsearch-es-http 9200:9200 &
PASSWORD=$(kubectl get secret elasticsearch-es-elastic-user -n elastic-system -o go-template='{{.data.elastic | base64decode}}')

# Vérifier shards non alloués
curl -k -u "elastic:$PASSWORD" https://localhost:9200/_cat/shards?v | grep UNASSIGNED

# Réallocation forcée
curl -k -u "elastic:$PASSWORD" -X POST \
  https://localhost:9200/_cluster/reroute?retry_failed=true

pkill -f "port-forward.*9200"
```

### Kibana inaccessible

```bash
# Vérifier Ingress/Load Balancer
kubectl get ingress -n elastic-system
kubectl describe ingress -n elastic-system

# Vérifier certificat SSL (si domaine configuré)
kubectl get managedcertificate -n elastic-system
kubectl describe managedcertificate -n elastic-system

# Port-forward en fallback
kubectl port-forward -n elastic-system svc/kibana-kb-http 5601:5601
# Puis ouvrir https://localhost:5601
```

**Causes fréquentes** :
1. DNS pas encore propagé (jusqu'à 48h)
2. Certificat SSL en cours de provisioning (15-30 min)
3. Firewall bloquant le port 443

### Fleet Server ne démarre pas

```bash
# Vérifier logs
kubectl logs -n elastic-system -l app=fleet-server --tail=100

# Vérifier connexion Elasticsearch
kubectl exec -n elastic-system -it $(kubectl get pod -n elastic-system -l app=fleet-server -o jsonpath='{.items[0].metadata.name}') -- \
  curl -k https://elasticsearch-es-http:9200

# Recréer Fleet Server
kubectl delete -n elastic-system -l app=fleet-server
./scripts/deploy.sh  # Redéploie uniquement Fleet
```

### Espace disque insuffisant

```bash
# Phase 1 : 100GB par node = 300GB total
# Si >80% utilisé, nettoyer ou migrer Phase 2

# Vérifier utilisation
kubectl exec -n elastic-system elasticsearch-es-default-0 -- \
  df -h /usr/share/elasticsearch/data

# Nettoyer anciens indices (>30 jours)
PASSWORD=$(kubectl get secret elasticsearch-es-elastic-user -n elastic-system -o go-template='{{.data.elastic | base64decode}}')
kubectl port-forward -n elastic-system svc/elasticsearch-es-http 9200:9200 &

curl -k -u "elastic:$PASSWORD" -X DELETE \
  "https://localhost:9200/logs-*-$(date -d '30 days ago' +%Y.%m)*"

pkill -f "port-forward.*9200"
```

**Recommandation** : Si >80% disk, migrer vers Phase 2 (580€/mois, 1 TB storage)

## 💡 Astuces Phase 1

### Optimiser les performances

```bash
# Augmenter refresh interval (moins de I/O)
PASSWORD=$(kubectl get secret elasticsearch-es-elastic-user -n elastic-system -o go-template='{{.data.elastic | base64decode}}')
kubectl port-forward -n elastic-system svc/elasticsearch-es-http 9200:9200 &

curl -k -u "elastic:$PASSWORD" -X PUT \
  "https://localhost:9200/_all/_settings" \
  -H 'Content-Type: application/json' \
  -d '{"index": {"refresh_interval": "30s"}}'

pkill -f "port-forward.*9200"
```

### Monitoring en temps réel

```bash
# Logs temps réel Elasticsearch
kubectl logs -n elastic-system -f -l elasticsearch.k8s.elastic.co/cluster-name=elasticsearch

# Logs Kibana
kubectl logs -n elastic-system -f -l kibana.k8s.elastic.co/name=kibana

# Logs Fleet
kubectl logs -n elastic-system -f -l app=fleet-server

# Utilisation ressources
watch -n 5 'kubectl top pods -n elastic-system'
```

### Backup manuel

```bash
# Créer snapshot manuel
PASSWORD=$(kubectl get secret elasticsearch-es-elastic-user -n elastic-system -o go-template='{{.data.elastic | base64decode}}')
kubectl port-forward -n elastic-system svc/elasticsearch-es-http 9200:9200 &

SNAPSHOT_NAME="manual-$(date +%Y%m%d-%H%M%S)"

curl -k -u "elastic:$PASSWORD" -X PUT \
  "https://localhost:9200/_snapshot/gcs-repo/$SNAPSHOT_NAME?wait_for_completion=false" \
  -H 'Content-Type: application/json' \
  -d '{"indices": "*", "ignore_unavailable": true}'

# Vérifier progression
curl -k -u "elastic:$PASSWORD" \
  "https://localhost:9200/_snapshot/gcs-repo/$SNAPSHOT_NAME"

pkill -f "port-forward.*9200"
```

### Accès shell Elasticsearch

```bash
# Shell dans pod Elasticsearch
kubectl exec -it -n elastic-system elasticsearch-es-default-0 -- /bin/bash

# Une fois dans le pod :
# - Config : /usr/share/elasticsearch/config/elasticsearch.yml
# - Data : /usr/share/elasticsearch/data
# - Logs : /usr/share/elasticsearch/logs
```

## 🔄 Mise à jour Phase 1

### Update Elasticsearch version

```bash
# Phase 1 : Rolling update automatique (zero downtime)
kubectl patch elasticsearch elasticsearch -n elastic-system \
  --type=merge -p '{"spec":{"version":"8.12.0"}}'

# ECK gère le rolling update automatiquement
# Durée : ~15 min (1 node à la fois)

# Suivre la progression
kubectl get elasticsearch -n elastic-system -w
```

### Scale Kibana (HPA)

```bash
# Kibana scale automatiquement 1-2 replicas en Phase 1
# Forcer scale manuel si besoin :

kubectl scale deployment kibana-kb -n elastic-system --replicas=2

# Retour auto-scale
kubectl autoscale deployment kibana-kb -n elastic-system \
  --min=1 --max=2 --cpu-percent=70
```
